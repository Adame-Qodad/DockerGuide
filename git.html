Guide complet d'utilisation de Git

1. Introduction à Git

Git est un système de gestion de version distribué permettant de suivre l'historique de vos projets et de collaborer efficacement avec d'autres développeurs.

2. Installation de Git

Windows

  1. Télécharger l'installateur depuis le site officiel : Git for Windows
  2. Exécuter l'installateur et suivre les instructions

macOS

brew install git

Linux

sudo apt update
sudo apt install git

3. Concepts clés

  • Dépôt (Repository) : L'endroit où est stocké l'historique du code.
  • Commit : Un snapshot des modifications.
  • Branche (Branch) : Une version parallèle du projet pour développer de nouvelles fonctionnalités.
  • Merge : L'intégration des modifications d'une branche dans une autre.
  • Remote : Un dépôt distant (ex : GitHub, GitLab) permettant la collaboration.

4. Commandes Git essentielles

# Cloner un dépôt
git clone <url_du_dépôt>
# Ajouter des fichiers au staging
git add <fichier>  ou  git add .
# Valider les modifications
git commit -m "Message du commit"
# Envoyer les commits vers le dépôt distant
git push
# Récupérer les modifications du dépôt distant
git pull

5. Gestion des branches

# Créer une nouvelle branche
git branch nouvelle_branche
# Basculer sur une branche
git checkout nouvelle_branche
# Créer et basculer sur une nouvelle branche
git checkout -b nouvelle_branche
# Fusionner une branche dans la branche courante
git merge branche_a_fusionner

6. Commandes avancées

# Voir l'historique des commits
git log
# Vérifier l'état du dépôt
git status
# Annuler des modifications mises en scène
git reset <fichier>
# Annuler un commit (soft reset)
git reset --soft HEAD~1
# Revenir à un commit spécifique
git checkout <commit_id>

7. Bonnes pratiques et conseils

  • Utiliser des messages de commit clairs et explicites.
  • Créer une branche pour chaque nouvelle fonctionnalité ou correction.
  • Effectuer des commits fréquents pour un historique détaillé.
  • Utiliser un dépôt distant (ex : GitHub) pour la sauvegarde et la collaboration.

8. Conclusion

Git est un outil puissant et flexible pour la gestion de version. Ce guide couvre les bases et quelques commandes avancées pour vous aider à démarrer. Continuez à explorer et à pratiquer pour tirer pleinement profit de Git.
